How do I add my own scripts?

  1. Duplicate the __stub.js file inside the /src/algos/scripts directory and rename it.
  2. In the /src/algos/index.js, copy/paste one of the existing entries. Change object key, path to the algorithm script and displayName to suit your algorithm. It should look something like this:
                            
algo_name: {
    worker: new Worker(new URL("./scripts/algo_name.js", import.meta.url)),
    displayName: "My algorithm's name",
},
    Unfortunately, due to how webpack imports webworkers - the paths to scripts must be written out at build-time and we can't map algorithm names to paths at run-time. If you know how to solve this problem - be sure to submit a PR :).

What information does the framework provide? What is the script expected to return?

The framework provides the maze dimensions, start and end position coordinates and an array with coordinates of the walls.

Your script should return an array path coordinate pairs [row, column], which will be drawn as the path in the maze.

There's a __stub.js file in the /src/algos/scripts folder. Copy it for your own script and use it as a guide for what your script accepts and returns.

There's also a sample algorithm (mrgd.js) written by Aleksandr Kozlov.

I'm getting a security error when I try initiating pathfinding.

You can't load web workers when running scripts from a local file. Serve the page through a web-server (like the "Live Server" plugin for VS Code).
